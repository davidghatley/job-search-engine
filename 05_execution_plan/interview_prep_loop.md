# Interview Prep Loop

## Weekly System
1. **Review**: 3 job descriptions → extract skills + keywords.
2. **Practice**: 2 SQL drills + 1 data modeling exercise.
3. **Design**: 1 internal tool/system design prompt.
4. **Debug**: 1 “bug hunt” session in a small repo.
5. **Story bank**: add 2 STAR stories (ownership, impact).

## Minimal Syllabus
- SQL: joins, window functions, CTEs, aggregation pitfalls.
- Data modeling: dimensional vs operational models.
- System design: ingestion → queue → worker → destination.
- Integration patterns: retries, idempotency, dead‑letter queues.

## Feedback Loop
- Log mistakes and rewrite answers.
- Update portfolio to address recurring gaps.
- Track improvements weekly.
